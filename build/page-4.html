Что такое Ноды и для чего они нужны?

Нода - это узел в децентрализованной сети, представляет собой программный комплекс в виде модулей. Нода устанавливается на ПК или сервер владельца, интегрируется с клиентской базой, контактами, аналитическими системами, может собирать информацию из публичных источников, находить необходимую информацию на нодах экосистемы.
Приемка данных
Приемка - место, куда приходят данные из разных источников. Там пчелы-приемники их маркируют, согласно структуре поступившей информации
Юридический отдел
Каждый источник проверяются пчелами‑юристами на CCPA + GDPR compliance. Если проверку проходят, то данные идут дальше по алгоритму
Обработка данных
В отделе обработки данных происходит: обработка, сортировка, агрегирование, вычисление. Исправление ошибок в указание данных.
Аналитический отдел
Этот отдел занимается посторением связей между данными, определение схожих данных, дубликатов исправление данных, определение типов данных.

После аналитического этапа пчелы отдела сегментации разделяют информацию по типам: люди, компании, услуги, сайты, товары, услуги.
Хранилище
После сегментации данные помещаются в чистое хранилище с протоколом Data Transfer, для передачи данных по APL. Тут же происходит шифрование данных и получение кодированных хешей
Банк
Банк хранит свой баланс и отвечает за транзакции. В его в ячейках хранятся ключи, ключ отправляют, деньги принимают и в хранилище прячут. Если другой пасечник хочет купить, то деньги платят, и ключ принимают.
Стражники
Весь улей защищают пчелы‑стражники. Они летают и охраняют все, а отделам хранения и банку они уделяют особое внимание

Коммуникация между нодами экосистемы

Ноды экосистемы связаны между собой по протоколам передачи данных. Экосистема может включать неограниченное количество нод, поставщиков данных и постоянно обновляется и расширяется.

Владелец ноды решает самостоятельно, с кем коммуницировать, кому будет доступна информация.

Какие типы Нод существуют в экосистеме?

Master Nodes

  • Функцонал MiniNode + SlaveNode + ...
  • Верификация транзакций Core, получение комиссии
  • Функционирование сервисов платформы
  • Обработка данных при помощи нейросетей
Баланс для функционирования: 1400000 HIV

Standart Nodes

  • Функцонал MiniNode + ...
  • Обеспечение функционирования сервисов Core
  • Подключение постащиков данных: компании, аналитические сервисы
  • Верификация P2P-транзакций, получение комиссии
  • Сложные операции обработки данных, поиск связей
Баланс для функционирования: 750000 HIV

Mini Nodes

  • Хранение своих данных c индексацией и поиском
  • Подключение постащиков данных: физ. лица, отзывы
  • Подключение аналитики для сайта для анализа посетителей
  • Сбор данных из различных источников
  • Прокси для анонимного доступа к ресурсам
Баланс для функционирования: 150000 HIV
